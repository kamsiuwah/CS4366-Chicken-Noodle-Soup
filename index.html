Welcome to Smart City Environmental Monitoring System

The Smart City Environmental Monitoring System is a web-based application that leverages IoT sensors to gather and analyze real-time data on air quality, temperature, humidity, wind speed, pressure and weather condition across the city. The system features an interactive dashboard for data visualization, custom alerts for abnormal conditions, and predictive analytics for forecasting environmental trends. Additionally, citizens can report environmental issues, contributing to a collaborative effort in ensuring urban environmental health. The platform is designed for scalability, security, and high performance.

Purpose

The Smart City Environmental Monitoring System has been developed to analyze environmental data such as aair quality, temperature, humidity, wind speed, pressure in real time. This system aims to help city planners and the citizens with issues related to monitoring environmental health and enable incitive prompt actions on raising concerns or taking up relevant actions over environmental issues.

Scope

The system will gather information through a distributed IoT sensor network and then a cloud-based processing and visualization through a webpage dashboard, followed by forecasting the trends by using the historical data. It will be a custom alerts and citizen issues reporting system.

Installation Guide

This guide provides step-by-step instructions to set up the Smart City Monitoring System for both the backend and frontend. Follow these steps to install the necessary components and dependencies.

Prerequisites

Before starting, ensure the following software is installed on your machine:

  • Python (version 3.7 or later)
  • Node.js (version 14 or later)
  • MongoDB Compass (for database management)

Backend Installation (Python)

  1. Install Python:
    • Download and install Python from the official website: https://www.python.org/downloads/.
    • Verify the installation by running in terminal: 
      python --version
  2. Navigate to the Backend Directory:
    • Open the main project directory, named smart city monitoring.
    • Inside, find the folder named backend. This folder contains all the code related to the backend.
  3. Install Dependencies:
    • In the backend folder, locate the file named requirements.txt. This file contains a list of all necessary packages.
    • Open a terminal in this directory and run the following command to install the dependencies: 
      pip install -r requirements.txt
  4. Database Configuration:
    • Ensure MongoDB is installed and running on your machine.
    • Configure your MongoDB connection string in the backend configuration files as needed for the project.

Frontend Installation (React)

  1. Install Node.js:
    • Download and install Node.js from https://nodejs.org/.
    • Verify the installation by running: 
      node --version
  2. Install MongoDB Compass:
    • Download and install MongoDB Compass from https://www.mongodb.com/products/compass.
    • MongoDB Compass allows you to manage the database for the application and visualize data stored.
  3. Install Frontend Dependencies:
    • In the main project directory, open a terminal and run: 
      npm install
    • This command installs all packages necessary to run the frontend of the application.

User Manual

This manual provides instructions to start the Smart City Monitoring System application, covering the key features and functionality.

Starting the Application

  1. Starting the Backend:
    • In the main project directory, navigate to the backend folder.
    • Open the backend directory in Visual Studio Code.
    • Open a terminal in VS Code and run the following command to start the backend server: 
      python app2.py
  2. Starting the Frontend Server:
    • In the main project directory, open a terminal in VS Code.
    • Navigate to the server directory by running: 
      cd server
    • Run the following command to connect the frontend server with MongoDB: 
      node server.js
  3. Starting the Frontend Application:
    • In the main project directory, open a new terminal and run: 
      npm start
    • This command starts the frontend of the application on the URL http://localhost:3000.
  4. Accessing the Application:
    • Open a web browser and go to http://localhost:3000 to access the landing page of the application.

Application Features

  1. Signup and Login:
    • On the landing page, there are options to Sign Up or Log In.
    • Sign Up: Users create an account by entering their credentials.
    • Log In: Users verify their credentials to access the main dashboard.
  2. Dashboard:
    • Once logged in, users are directed to the Dashboard, where they can view:
    • Current Weather Conditions: Temperature, humidity, pressure, wind speed, and air quality index.
    • 24-Hour Data Graphs: Graphs showing weather trends over the past 24 hours.
  3. Predictive Analysis:
    • In this section, users can view predicted weather conditions for the next 24 hours displayed in graphical form.
  4. Alerts:
    • Users can access alerts for any harsh weather conditions.
  5. Report an Issue:
    • In this section, users can report issues related to weather by sending an email directly to an environmental agency.
  6. About Us:
    • This section provides information about the project and its purpose.
  7. Settings:
    • Profile Settings: Users can update their profile information.
    • Alert Settings: Users can set threshold values for receiving alerts about specific weather conditions.
  8. Logout:
    • The Logout button allows users to securely exit the application.
